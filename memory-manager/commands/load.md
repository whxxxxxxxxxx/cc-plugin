# 记忆加载

参数：$ARGUMENTS

## 操作步骤

### 情况一：未提供参数

列出所有可用记忆文件供用户选择：

1. 列出 `./.claude/sessions/` 中符合的所有文件，按创建时间降序排列
2. 以编号列表展示，格式如下：

```
可用的记忆文件：

  1. 2026-02-27-some-topic.md
  2. 2026-02-20-another-topic.md
  ...

请输入编号（多个用逗号分隔，如 1,3）或直接输入文件名，留空则加载最近 7 天：
```

3. 等待用户输入：
   - 输入编号（如 `1` 或 `1,3`）：加载对应文件
   - 输入文件名：直接加载该文件
   - 直接回车（空输入）：加载最近 7 天内的所有文件

### 情况二：提供了参数

参数为逗号分隔的文件名列表，例如 `2026-01-01-topic.md,2026-01-02-other.md`

1. 通过逗号分割并去除空格来解析 $ARGUMENTS
2. 对每个文件名，读取 `./.claude/sessions/<filename>`
3. 显示每个文件的内容

## 输出格式

对于每个加载的记忆文件，显示：

```
### <filename>
<文件内容>
---
```

加载所有文件后，总结："已加载 N 个记忆文件。" 并简要列出加载的主题。

如果文件不存在，警告用户："文件未找到：`.claude/sessions/<filename>`"
